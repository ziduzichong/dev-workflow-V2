> 适用平台：OpenCode（优先）/ Claude Code（兼容）/ 相近 agent 平台

## 一、文件结构

```
dev-workflow-v2/
├── README.md              ← 本文档
├── INSTALL.md             ← 安装指南
└── skills/
    └── dev-workflow-v2/
        └── SKILL.md       ← 唯一核心文件（~600行，完全自包含）
```

## 二、与 V1（Claude Code 插件版）的对比

| 对比维度 | V1 | V2 |
|---------|----|----|
| **依赖方式** | 调用 10 个外部 superpowers skill | **全部内置**，无需安装任何插件 |
| **文件结构** | SKILL.md + dependencies.json + 10个 references 文件 | **单一 SKILL.md**，所有逻辑自包含 |
| **平台** | Claude Code 专用（依赖 Skill/Agent/TaskCreate 等专属工具） | OpenCode 原生 + 兼容 Claude Code |
| **安装** | 需安装 superpowers 插件 + npx skills | 复制一个目录即可 |
| **文件大小** | ~720 行 + 10×~200行外链 = ~2700 行 | ~600 行单文件 |
| **维护成本** | 改阶段需同步多个文件 | 单文件内段落定位修改 |
| **Context 消耗** | 按需加载（较小） | 一次加载 ~600 行（中等） |

## 三、为什么要自包含？

**核心问题：superpowers 插件是 Claude Code 专属的。**

| 依赖 | V1 | V2 |
|------|----|----|
| brainstorming（需求分析/方案对比） | `Skill` 调用外部 skill | **内联** 苏格拉底追问 + 方案模板 |
| writing-plans（实现计划分解） | `Skill` 调用外部 skill | **内联** 任务拆分模板 |
| TDD 循环 | `Skill` 调用外部 skill | **内联** 红绿重构循环逻辑 |
| systematic-debugging | `Skill` 调用外部 skill | **内联** 7步调试流程 |
| code-review | `Skill` 调用外部 skill | **内联** 5维度审查清单 + 任务表 |
| code-simplifier | `Skill` 调用外部 skill | **内联** 3维度优化 + lint 命令 |

V2 把每个依赖的**核心方法论**（不是全部功能）重写并进 SKILL.md，确保在多平台环境下无需外部依赖即可完整执行。

## 四、设计决策

### 决策1：单文件 vs 多文件

| 方案 | 优点 | 缺点 | 选择 |
|------|------|------|------|
| **单文件** | 安装简单，逻辑一定被加载 | 文件较大，修改需全文定位 | ✅ 采用 |
| 多文件 | 结构清晰，可按需跳转 | AI 可能忘记加载额外的文件 | ❌ |

**理由：** 实测中 AI 偶尔会忘记手动 Read 额外文件，导致执行不完整。单文件确保所有逻辑一定被加载。

### 决策2：保留全部 V1 方法论

V2 完整保留了 V1 经过 6 轮审查迭代确立的方法论：
- 50 行法则（智能流程裁剪）
- 苏格拉底追问（目标/边界/约束/验收）
- 微任务拆分（2~5 分钟粒度含文件路径+步骤+验证命令）
- TDD 红绿重构循环 + 嵌入式降级方案（cppcheck/ruff）
- 阶段3→4 桥接检查（TDD闭合/构建日志/TODO残留）
- 证据强制规则（所有检查点须附带工具输出）
- 中止报告标准格式（含恢复指引）
- 遥测 JSON 格式（每阶段耗时/重试/通过率）

### 决策3：OpenCode 优先但兼顾其他平台

```yaml
compatibility: opencode    # 主平台声明
```

- OpenCode 可直接作为 agent 加载此 SKILL.md
- Claude Code 也能读取（忽略不认识的 frontmatter 字段 `compatibility`/`version`）
- 其他 agent 平台只要支持 Markdown 指令注入即可使用

## 五、优缺点分析

### ✅ 优点

| 优点 | 说明 |
|------|------|
| **完全自包含** | 无需安装任何外部插件，下载即用 |
| **OpenCode 原生** | 严格遵循 OpenCode SKILL.md 规范 |
| **继承 V1 全部方法论** | 50行法则、苏格拉底追问、微任务、桥接检查、遥测等 |
| **适合嵌入式开发** | 内置 C/C++ 静态分析降级方案 |
| **单文件易安装** | 一个 SKILL.md 放对位置即可工作 |
| **方法论内联** | 所有 7 阶段操作步骤直接可执行，不需跳转 |

### ❌ 缺点

| 缺点 | 缓解方式 |
|------|---------|
| 文件 ~600 行，一次性加载 | 现代 agent context 窗口通常 200K+ tokens，实际影响小 |
| 不如多文件结构方便维护 | 用 `## 阶段N` 标题快速定位 |
| 无法自动同步 superpowers 更新 | superpowers 更新频次低，手动同步可接受 |
| 非可视化进度追踪 | 通过遥测 JSON 文件查看进度 |

## 六、推荐使用场景

| 场景 | 推荐版本 |
|------|---------|
| 主要用 OpenCode | **V2**（原生兼容） |
| 主要用 Claude Code + superpowers 完整版 | V1（功能更全） |
| 两个工具都使用 | **V2**（两者都兼容） |
| 需要最简化安装 | **V2**（单文件，零依赖） |
| 嵌入式/STM32 开发 | **V2**（内置降级方案） |
